# Notari — Product Planning

This folder captures the product-planning artifacts for Notari, following Maestro's documented phase structure (Service Vision → Problem Brief → Success Metrics → Solution Spec → User Stories → UI Spec → Sprint Roadmap).

These are **living documents** — they reflect intent at the time of writing and should be updated as decisions evolve. Sprints execute against them.

## Files

| Phase | File | Purpose |
|-------|------|---------|
| 0 | [`service-vision.md`](./service-vision.md) | What Notari is and the long-term vision |
| 1.1 | [`problem-brief.md`](./problem-brief.md) | The problem we're solving and for whom |
| 1.3 | [`success-metrics.md`](./success-metrics.md) | Target users + how we measure success |
| 2.1 | [`solution-spec.md`](./solution-spec.md) | Full feature spec and technical decisions |
| 2.2 | [`user-stories.md`](./user-stories.md) | Functionality framed as user-facing stories |
| 2.3 | [`ui-spec.md`](./ui-spec.md) | Screen inventory + platform breakpoint behavior |
| 3 | [`sprint-roadmap.md`](./sprint-roadmap.md) | Epics, sprint sequence, and release versioning |

## Open decisions

Items marked `❓ TBD` in the documents need user input before the affected sprints can begin. The most urgent ones are surfaced in `sprint-roadmap.md`.
